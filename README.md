### 项目整体介绍
这是一个**基于多模态技术的视频问答与检索系统**，核心目标是接收用户的文本查询，从视频库中精准检索相关视频，并结合视频的视觉内容、音频转写文字、文本描述等多维度信息，给出符合查询意图的问答结果，同时支持合规的视频播放交互。

系统整合了计算机视觉、语音识别、大语言模型（多模态/重排序/嵌入）、检索引擎、Web服务等技术，实现“检索-问答-交互”全流程的视频语义理解与问答能力。

### 一、核心技术栈
| 技术领域       | 核心组件/工具                                                                 |
|----------------|------------------------------------------------------------------------------|
| 编程语言       | Python                                                                       |
| 视觉处理       | OpenCV（关键帧提取）、PIL（图像处理）、CLIP（跨模态特征对齐）                  |
| 音频处理       | moviepy（音频提取）、pydub（音频切割）、Whisper（ASR语音转文字）               |
| 大模型         | Qwen2.5-VL（多模态问答）、Qwen3-Reranker（检索重排序）、Qwen3-Embedding（文本嵌入） |
| 检索引擎       | Whoosh（TF-IDF文本检索）、FAISS（向量检索）                                  |
| 中文处理       | jieba（中文分词）                                                             |
| 深度学习框架   | PyTorch、HuggingFace Transformers（模型推理）                                |
| Web服务        | Flask（后端API+前端交互）                                                     |

### 二、核心模块与文件功能
系统按“数据预处理（特征入库）”和“在线问答检索”分为两大阶段，各文件功能如下：

#### 1. 数据预处理/特征入库阶段（核心：`extract_feature.py`）
该阶段完成视频库的特征提取、索引构建，为在线检索做准备：
- `extract_frame.py`：提取视频关键帧——基于帧间灰度差异筛选TopK帧（帧差越大代表画面变化越剧烈，是视频核心视觉信息）；
- `clip_feature.py`：封装CLIP模型，提取图像（关键帧/图片）、文本的归一化向量特征，用于跨模态检索；
- `convert_mp42wav.py`：将MP4视频的音频轨道提取为WAV格式；
- `extract_audio_text.py`：切割WAV音频+调用Whisper模型做ASR，将音频转为文字；
- `describe.py`：调用Qwen2.5-VL多模态模型，生成视频/图片的文本描述；
- `qwen3_embedding.py`/`Embedding.py`：分别封装Qwen3-Embedding、BERT模型，生成文本（视频描述/音频文字）的归一化嵌入向量；
- `extract_feature.py`（主脚本）：遍历视频库，整合上述模块，完成：
  - 视觉特征入库：关键帧CLIP向量存入FAISS的`img_index`；
  - 文本特征入库：视频描述/音频文字的嵌入向量存入FAISS的`text_index`；
  - 文本检索入库：基于jieba分词构建Whoosh的TF-IDF索引（存储视频ID、文本描述、音频文字）；
  - 存储索引文件、路径映射字典（关联向量索引ID与视频路径）。

#### 2. 在线问答检索阶段（核心：`app.py`）
该阶段通过Flask提供Web服务，响应用户查询，核心流程：
- `cal_score.py`：封装Qwen3-Reranker模型，对“用户查询-候选回答”对打分，实现重排序；
- `app.py`（主脚本）：
  1. 接收用户文本查询；
  2. **多路召回**：
     - TF-IDF召回：Whoosh检索分词后的查询，匹配相关视频；
     - CLIP召回：查询文本转CLIP向量，检索FAISS的`img_index`，匹配视觉相似视频；
     - 文本嵌入召回：查询转Qwen3-Embedding向量，检索FAISS的`text_index`，匹配语义相似视频；
  3. **结果融合**：对三路召回的视频ID做分数加权求和，取Top3候选；
  4. **多模态问答**：对每个候选视频，调用Qwen2.5-VL结合“查询+视频+音频文字”生成回答；
  5. **重排序**：Qwen3-Reranker对“查询-回答”打分，取最高分回答，返回回答内容+视频路径；
  6. **视频播放**：提供合规的视频播放API（仅允许指定目录下的视频，避免安全风险）。

---

### 三、项目核心亮点
1. **多模态融合**：整合视频视觉（关键帧）、音频（ASR文字）、文本（描述/嵌入）多维度信息，提升问答/检索的全面性；
2. **多路召回+重排序**：TF-IDF（文本匹配）+ CLIP（跨模态匹配）+ 文本嵌入（语义匹配）保障召回覆盖率，Reranker提升结果精准度；
3. **大模型赋能**：基于Qwen系列大模型（多模态/重排序/嵌入）实现高精度的语义理解与问答；
4. **工程化设计**：预处理与在线检索分离，FAISS/Whoosh保障检索效率，Flask提供友好的Web交互；
5. **安全管控**：视频播放仅允许指定目录，避免路径遍历等安全风险。

---

### 四、典型使用场景
用户输入自然语言查询（如“视频里的摩托车是什么品牌？”“这段音频里提到了什么政策？”），系统能：
1. 从视频库中检索相关视频；
2. 基于视频的视觉、音频文字生成精准回答；
3. 返回回答结果+可播放的视频路径，支持用户核验。
